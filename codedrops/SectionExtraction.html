Requirements

  • Parse and separate sections of a CodeDrop into distinct components (files).
  • Each section must retain its formatting and include a documentation header indicating its origin.
  • Support extraction for all common sections: Requirements, Design, Implementation, Testing, and Documentation.
  • Provide an API to extract sections programmatically.

Design

Component: CodeDropExtractor

Inputs: Single CodeDrop file (e.g., HTML, Markdown)

Outputs: Set of files categorized by section type with appropriate documentation headers.

Design Steps:

  • Identify section markers using regular expressions.
  • Extract section content between markers.
  • Save each section as a separate file with headers indicating the section type and source file.
  • Provide error handling for missing sections or malformed input formats.
